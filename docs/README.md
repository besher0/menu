# Restaurant Menu Builder SaaS — Complete Reference Pack

This documentation pack is written as the main product, architecture, and development reference for a multi-tenant Restaurant & Café Menu Builder SaaS platform.

The goal is not to describe a simple CRUD menu application. The goal is to define a scalable SaaS product similar in spirit to a specialized combination of Shopify, WordPress, Framer, and a digital restaurant ordering/menu platform.

The platform allows restaurants and cafés to:
- Build their own public menu websites.
- Customize the website design without code.
- Manage products, categories, branches, media, QR codes, analytics, SEO, PWA behavior, and WhatsApp ordering.
- Unlock advanced capabilities through subscription plans.
- Use 3D and VR/360 product views when enabled.
- Use white-label branding and custom domains in higher plans.

## Recommended Usage With Codex

Place this entire folder inside your project as:

```txt
docs/
```

Then tell Codex:

```txt
Read the docs folder first. Treat these documents as the source of truth.
Do not start coding before understanding:
- architecture
- database design
- feature flags
- plugin system
- subscription model
- menu builder
- theme engine
- public website
- PWA
- WhatsApp ordering
- 3D/VR
- roadmap
```
